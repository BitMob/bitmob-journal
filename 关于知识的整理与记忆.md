## <a name="collect"/></a>收集
### 场景 1：iOS

#### <a name="collect-1"/></a>方案 1: 捷径 + Evernote
以生词为例，遇到不认识的单词，先用 iOS 自带的词典查阅一次意思，然后 copy，使用自定义Shortcut 将刚刚复制的内容添加到 Evernote 的 New words 笔记本。演示如下：

![](./assets/collect-new-word-ios-1.gif)

一键收集剪贴板到 Evernote 的捷径配置如下图：

![](./assets/Shortcuts-screenshot.jpeg)

<br />

#### Tips
- 操作里面查单词的步骤不是必要的，但为了能更顺利的阅读下去，先让自己了解一下单词的意思。
- 当然你也可以在捷径中设置成其他笔记应用，比如 Bear。但我自己试过，收集到 Evernote 是最快的，收集到 Bear 会产生跳转动作，打断阅读。
- 不需要等「捷径」完成，就可以上滑关闭「小工具」。
- 除了演示中的收集单词，也可以利用「捷径」和 Evernote 收集网络文章、新闻段落，甚至增加一步「立即打开该笔记」记录下当前的想法。

<br />

#### <a name="collect-2"></a>方案 2: Pin (或 Copied / Paste 2) + Evernote
还是以 iOS 的剪贴板为基础，将剪贴板内的内容先利用 Pin (或其他剪贴板管理 App 比如 Copied 或 Paste 2) 的小工具逐个将单词收集起来，然后利用 Pin 的第三方键盘，将内容收集到 Evernote。演示：

![](./assets/collect-new-word-ios-2.gif)

#### Tips
- 因为 iOS 禁止 App 在非激活状态下获取剪贴板内容，所以每 copy 一个/一段文字，需要打开小工具面板，让 Pin 等应用处于激活状态，获取最近的剪贴板内容。

<br />

#### 两个方案对比
- **方案 1** 完全免费，**方案 2** 中的 Pin 需要 6 元购买。
- **方案 1** 比**方案 2** 多了一次点击「捷径」的动作。
- **方案 1** 中的「捷径」因为指定了某个笔记，所以能够明确预期，每次复制后，点击「捷径」，目标笔记内的所收集内容，都是相同性质的；而**方案 2** 由于剪贴板历史可能并不所有都是需要收集到同一本笔记的内容，所以到后期批量录入的时候需要费点神区别一下。

<br />

### 场景 2: macOS
#### <a name="collect-macos-1"></a>方案 1: Evernote 快捷笔记
在 Mac 打开 Evernote 你会见到屏幕顶部菜单栏右方的大象图标，点击它或者使用快捷键 `⌘ + ctrl + N` 会弹出一个快速笔记的窗口，这就是用来快速录入你在电脑前想收集的任何信息，包括但不限于：生词、文字段落、网址、图片等信息。在这里面输入的一切只是临时的，直到按下窗口下方的「保存到 Evernote」，才会将窗口内的所有内容新建成一份独立的笔记，此后快速笔记窗口的内容将被清空。

![](./assets/collect-new-word-macos-1.jpg)

<br />

#### <a name="collect-macos-2"></a>方案 2: Alfred Evernote Workflow
Alfred 可以说是 Mac 用户必备装机 App，免费功能包括：快速启动程序、自定义搜索、快速查字典、快速计算，付费开通 PowerPack (英镑 £19) 后即可使用与 Pin 类似的快速获取剪贴板历史，以及现在要介绍的、极其强大的自定义 Workflows。

Alfred Workflows 像是更强大的 iOS 的「捷径」，用户可以自己编写脚本，我自己就做了很多脚本用来快速打开工程（VS Code + Chrome + npm scripts + Finder 目录），[官方论坛](https://www.alfredforum.com/forum/3-share-your-workflows/)以及[第三方论坛](http://www.packal.org)还提供很多别人写好的 Workflows，包括 Evernote 官方开发的。先[下载](https://www.dropbox.com/s/zkhlk4b1ri9tava/Evernote8_992.alfredworkflow?dl=0)，双击安装到Alfred，成功后可以在 Alfred 的设置面板见到：

![](./assets/collect-new-word-macos-1.jpg)

这里有很多快速 Evernote 动作，包括搜索笔记、创建待办事项、创建提醒、创建笔记等。这里我只介绍如何用这套 workflow 快速将文字附加到指定笔记，其他功能请自行探索。
1. 唤出 Alfred (我设置的快捷键是 `⌘ + Space`)
2. 键入 `enn`，会弹出三个选项：用所选内容创建笔记、用剪贴板内容创建笔记、手动输入并创建笔记。
3. 此时用上下键选择一项，按住 `⌘`，便会从「创建」模式变为「附加」模式，按下回车，会弹出一个最近使用的笔记清单，选择其中一个按回车，就可以将所选内容 / 剪贴板内容附加到目标笔记。
演示：

![](./assets/collect-macos-evernote-workflow-screenshots.gif)

<br />

#### <a name="collect-macos-3"></a>方案 3: Alfred PowerPack (或 Pin) + Evernote
与 Pin 等剪贴板扩展 App 类似，Alfred 也有剪贴板历史，与 iOS 不同的是，macOS 允许应用在背景获取剪贴板内容，因此你可以狂按 `⌘ + C` 不断保存剪贴板历史。
先在 Alfred 设置好「弹出剪贴板历史」的全局快捷键：
![](./assets/collect-new-word-macos-2.jpg)
按下后效果如图：
![](./assets/collect-new-word-macos-3.jpg)
现在打开 Evernote 的相应笔记，弹出 Alfred 的剪贴板历史，按下 `⌘ + n(旁边的数字)` 粘贴需要的内容。

#### Tips
- 如果觉得 Alfred 的 PowerPack 太贵 (英镑 £19)，可以尝试 Pin 的 macOS 版 (30元)，本人因为觉得功能重复，尚未购入，未能提供使用体验对比，见谅。
- Alfred 剪贴板历史可以粘贴到任何输入框中，并非只能粘贴到笔记。
- 如果你同时在使用 iOS 和 macOS，并用同一个 iCloud 账号登录设备，剪贴板是会云同步的，即在 iOS 复制的内容会同步到所有设备，只要电脑开着 Alfred，所有设备的复制内容都会进入 Alfred 的剪贴板历史，也就是说，这种情况下，你可以在 iPhone 进行阅读、复制文本，便可以在电脑上查阅、使用剪贴板历史，而省略掉在 iOS 上使用「捷径」或 Pin 的步骤，非常方便。
- 利用剪贴板历史会出现的问题与 iOS 方案 2 类似，因为复制的内容性质可能不一致，后期统一采集的时候要稍作区分。
