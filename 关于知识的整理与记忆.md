## <a name="collect"/></a>收集
### 场景 1：iOS

#### <a name="collect-1"/></a>方案 1: 捷径 + Evernote
以生词为例，遇到不认识的单词，先用 iOS 自带的词典查阅一次意思，然后 copy，使用自定义Shortcut 将刚刚复制的内容添加到 Evernote 的 New words 笔记本。演示如下：

![][image-1]

一键收集剪贴板到 Evernote 的捷径配置如下图：

![][image-2]

<br />

#### Tips
- 操作里面查单词的步骤不是必要的，但为了能更顺利的阅读下去，先让自己了解一下单词的意思。
- 当然你也可以在捷径中设置成其他笔记应用，比如 Bear。但我自己试过，收集到 Evernote 是最快的，收集到 Bear 会产生跳转动作，打断阅读。
- 不需要等「捷径」完成，就可以上滑关闭「小工具」。
- 除了演示中的收集单词，也可以利用「捷径」和 Evernote 收集网络文章、新闻段落，甚至增加一步「立即打开该笔记」记录下当前的想法。
- \_11.20更新\_「捷径」配置中的「目前的日期」以及下方的分割线是用来更清晰地区分每次收集的内容，到整理的时候不会那么费劲。

<br />

#### <a name="collect-2"></a>方案 2: Pin (或 Copied / Paste 2) + Evernote
还是以 iOS 的剪贴板为基础，将剪贴板内的内容先利用 Pin (或其他剪贴板管理 App 比如 Copied 或 Paste 2) 的小工具逐个将单词收集起来，然后利用 Pin 的第三方键盘，将内容收集到 Evernote。演示：

![][image-3]

#### Tips
- 因为 iOS 禁止 App 在非激活状态下获取剪贴板内容，所以每 copy 一个/一段文字，需要打开小工具面板，让 Pin 等应用处于激活状态，获取最近的剪贴板内容。

<br />

#### 两个方案对比
- **方案 1** 完全免费，**方案 2** 中的 Pin 需要 6 元购买。
- **方案 1** 比**方案 2** 多了一次点击「捷径」的动作。
- **方案 1** 中的「捷径」因为指定了某个笔记，所以能够明确预期，每次复制后，点击「捷径」，目标笔记内的所收集内容，都是相同性质的；而**方案 2** 由于剪贴板历史可能并不所有都是需要收集到同一本笔记的内容，所以到后期批量录入的时候需要费点神区别一下。

<br />

### 场景 2: macOS
#### <a name="collect-macos-1"></a>方案 1: Evernote 快捷笔记
在 Mac 打开 Evernote 你会见到屏幕顶部菜单栏右方的大象图标，点击它或者使用快捷键 `⌘ + ctrl + N` 会弹出一个快速笔记的窗口，这就是用来快速录入你在电脑前想收集的任何信息，包括但不限于：生词、文字段落、网址、图片等信息。在这里面输入的一切只是临时的，直到按下窗口下方的「保存到 Evernote」，才会将窗口内的所有内容新建成一份独立的笔记，此后快速笔记窗口的内容将被清空。

![][image-4]

<br />

#### <a name="collect-macos-2"></a>方案 2: Alfred Evernote Workflow
Alfred 可以说是 Mac 用户必备装机 App，免费功能包括：快速启动程序、自定义搜索、快速查字典、快速计算，付费开通 PowerPack (英镑 £19) 后即可使用与 Pin 类似的快速获取剪贴板历史，以及现在要介绍的、极其强大的自定义 Workflows。

Alfred Workflows 像是更强大的 iOS 的「捷径」，用户可以自己编写脚本，我自己就做了很多脚本用来快速打开工程（VS Code + Chrome + npm scripts + Finder 目录），[官方论坛][11]以及[第三方论坛][12]还提供很多别人写好的 Workflows，包括 Evernote 官方开发的。先[下载][13]，双击安装到Alfred，成功后可以在 Alfred 的设置面板见到：

![][image-5]

这里有很多快速 Evernote 动作，包括搜索笔记、创建待办事项、创建提醒、创建笔记等。这里我只介绍如何用这套 workflow 快速将文字附加到指定笔记，其他功能请自行探索。
1. 唤出 Alfred (我设置的快捷键是 `⌘ + Space`)
2. 键入 `enn`，会弹出三个选项：用所选内容创建笔记、用剪贴板内容创建笔记、手动输入并创建笔记。
3. 此时用上下键选择一项，按住 `⌘`，便会从「创建」模式变为「附加」模式，按下回车，会弹出一个最近使用的笔记清单，选择其中一个按回车，就可以将所选内容 / 剪贴板内容附加到目标笔记。
演示：

![][image-6]

#### Tips
- 如果附加内容来自 Safari 或其他浏览器，Evernote 会自动加上网址，如果只是一个单词的话，后期整理会稍稍觉得不便。(当然，如果你会用 Sublime Text 等编辑器快速处理文本就另说)
- 从网页上粘贴的内容，会把网页的样式（颜色、字号、底色等）一并附上，收集的时候先不用管，到整理的时候打开笔记，按下 `⌘ + Shift + F` 就可以清除样式。


<br />

#### <a name="collect-macos-3"></a>方案 3: Alfred PowerPack (或 Pin) + Evernote
与 Pin 等剪贴板扩展 App 类似，Alfred 也有剪贴板历史，与 iOS 不同的是，macOS 允许应用在背景获取剪贴板内容，因此你可以狂按 `⌘ + C` 不断保存剪贴板历史。
先在 Alfred 设置好「弹出剪贴板历史」的全局快捷键：
![][image-7]
按下后效果如图：
![][image-8]
现在打开 Evernote 的相应笔记，弹出 Alfred 的剪贴板历史，按下 `⌘ + n(旁边的数字)` 粘贴需要的内容。

#### Tips
- 如果觉得 Alfred 的 PowerPack 太贵 (英镑 £19)，可以尝试 Pin 的 macOS 版 (30元)，本人因为觉得功能重复，尚未购入，未能提供使用体验对比，见谅。
- Alfred 剪贴板历史可以粘贴到任何输入框中，并非只能粘贴到笔记。
- 如果你同时在使用 iOS 和 macOS，并用同一个 iCloud 账号登录设备，剪贴板是会云同步的，即在 iOS 复制的内容会同步到所有设备，只要电脑开着 Alfred，所有设备的复制内容都会进入 Alfred 的剪贴板历史，也就是说，这种情况下，你可以在 iPhone 进行阅读、复制文本，便可以在电脑上查阅、使用剪贴板历史，而省略掉在 iOS 上使用「捷径」或 Pin 的步骤，非常方便。
- 利用剪贴板历史会出现的问题与 iOS 方案 2 类似，因为复制的内容性质可能不一致，后期统一采集的时候要稍作区分。

### <a name="collect-books"></a>场景 4: 实体书
实体书不能复制粘贴，也不大可能遇到哪个段落需要记录的就马上在手机或电脑上敲下来，既低效又打断阅读。最实际可行的方法是用 OCR App 进行文字识别。推荐使用「小嘿扫描」(中国 App Store ¥3.00 非常实惠)，通常一次扫描识别的流程是这样的：

1. 打开应用，从底部菜单选择「拍照」
2. 把你要摘抄的书页拍下来
3. 裁切你要的部分
4. 按右下角黄色按钮进行识别
5. 识别之后可以进行复制，然后使用上述的「复制插入到笔记」方法记录下来

但其实在读书的过程之中多次重复上述步骤也是很累（而且，打断阅读）的，你可以选择用手机自带拍照 App **先把书页全部都拍下来**，合上书之后，用「小嘿扫描」从相册里面选出你刚才拍的照片进行识别，它会自动把识别结果记录到「历史」列表，所有照片扫描之后，点击右上方「选择」，进行多选、合并文字、复制、插进笔记，最后删掉历史和照片。

#### Tips
- 「小嘿扫描」除了识别中文，还可以选择英语、日语、韩语、法语、德语、意大利语、西班牙语、俄语、葡萄牙语！没有广告，识别率高！赶紧点击左上方齿轮，请作者喝杯咖啡、打个好评吧！
- 对于竖排文字，「小嘿扫描」似乎没什么办法。

## <a name="sort"></a>整理及释义
这应该是知识积累最繁琐耗时、但是最重要的部分了。最好能每天抽出一整段时间坐在电脑前，专注于整理。面对乱糟糟的 Inbox，我的做法是：

1. 分类：新单词、新概念、新闻事件、文章摘录、读书笔记、冷知识、金句等等，并分别剪切到不同的笔记本，直到清空 Inbox。
2. 将所有条目整理成 `标题 + 释义` 的格式，方便之后生成学习卡片。
3. 较长的摘录内容（文章段落等）用一两句话压缩其内容，作为它的标题，帮助强化理解，比如：
	> **希腊与基督教如何看待德行与知識的关系**
	> 德行与知識之間这种密切的联系，乃是苏格拉底和柏拉图两人的特色。在某种程度上，它也存在于一切的希腊思想之中，而与基督教的思想相对立。在基督教的伦理里，内心的純洁才是本质的东西，并且至少是在无知的人和有学問的人之間同样地可以找得到的东西。希腊伦理学与基督教伦理学之間的这一区别，一直貫穿到今天。《西方哲学史》P172

4. 短的笔记，比如生词、新闻事件等，需要手动找到其释义或者详细内容，比如：
	> **imperative**
	> of vital importance; crucial: immediate action was imperative. 迫切的
	> giving an authoritative command; peremptory. 命令式的

## <a name="make-card"></a>制作卡片
如果玩过 Duolingo，就知道什么是记忆卡片了，[维基][14]上的定义是：
> 抽认卡或稱閃卡（英文：Flash Card）、字卡，是一小块纸片，在学校里用来作辅助教学。抽认卡可以用来记录词汇、历史事件时间、公式等等。使用抽认卡的目的主要是帮助记忆，提高學生的學習動機、以及在課堂上為學生提供多種感官的刺激。你在每个卡片上写下一个问题（同时在背面记下答案），用它们来测试自己，并根据你的测试、学习结果把它们进行排序、分组。
> 
> 这种策略使得学习更有选择性，也就是：在一组的卡片越难，你就越要花时间复习这一组。于是，你就能节省很多学习时间。

这种卡片尤其适合于生词的学习，但也不妨碍我们把它套用在其他类别的知识上，比如「世界最高的山峰」「各种艺术派别」「植物的名称」这样需要强记、条目之间没有太强的逻辑/因果关系的知识点。

市面上有很多制作记忆卡片的软件，这里不得不提到最被[学习狂人们][15]吹捧的电子化记忆卡片始祖 [Anki][16]，其功能强大、博大精深到令人发指，包括自定义样式、多媒体内容、根据[遗忘曲线][17]自动/自定义复习提醒等等。但是！如果不去刻意定制卡片样式，Anki 的界面实在是太简陋了，UI 设计约等于无，而且手机应用也比较贵（网页版和桌面版是免费的，这么复杂的系统只靠卖 App 赚钱，而且是一次性买断，其实也很良心了），如果要玩得出神入化，还得花很多时间把对照[说明书][18]实践，实在没那么多功夫。还好朋友推荐了我一款勉强可以接受的替代品 [Quizlet][19]。

Quizlet 网页版、手机 App 均免费使用基础班，大致的功能包括：
1. 自建「学习集 (decks)」，可以批量导入条目
2. 支持多种语言，还能发音，你自己输入的句子、文章都可以读出来
3. 多种记忆模式：
	- 单词卡（逐个翻看）
	- 学习（默写、选择题、判断题混合）
	- 书写（看释义默写）
	- 拼写（听读音默写）
	- 测试（自动生成一份试卷）
	- 配对（类似连连看小游戏）
	- 重力（限时陨石砸地球游戏）
4. 与朋友组建班级，一起学习、测试
5. 导入其他用户制作好的学习集

它的界面大概是这样的：
![][image-9]
![][image-10]

现在重点说说如何根据整理出来的笔记批量生成「学习集」。

我最近整理好的新单词在 Evernote 里面是这样拍版的：
![][image-11]
打开 Quizlet 网页版，点击页面顶部的`建立`按钮，输入标题，比如「Recent New Words」，然后点击`+ 自Word、Excel、Google文件等匯入」`，其实它并没有提供一个上传文件的功能，只是会出现一个文本输入框，现在把笔记里面的内容复制，粘贴到这个输入框，它就会根据框里面的内容先整理出一个预览。你会发现生成的结果并不是你想要的`单词 + 释义`的结构，因为它用了自己默认的分词方法: 用`Tab`来区隔单词和释义，用新的一行来区隔两张单词卡。

这种默认的区隔方法对于一个单词有多段释义的情况很不友善，还好 Quizlet 提供自定义区隔字符，请看截图：
![][image-12]
根据我的笔记排版，单词与释义之间是断行，两个单词之间（上面是上一个单词的释义，下面是新单词），用了一条小横线`— `，于是我就可以在 Quizlet 自定词语与释义之间写入`\n`，即一个换行符，在单词卡之间用`\n —\n`，即一个换行符加上一条横线再加上一个换行符。输入后，单词卡预览里显示的就是我所期待的结果了。当然你可以根据自己的喜好使用别的自定义区隔符。
